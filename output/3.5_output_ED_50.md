## Changes from 3.4

1. **Weak-pool ordering for downgraded TOC sections.** 3.4 introduced the TOC
   downgrade (a tier-2 section whose header names the clause but whose body is
   under 400 chars, or whose header ends in a page number, is demoted to tier 0).
   But demoted sections landed in the weak pool in plain document order, so a
   downgraded TOC entry could still crowd out real content within the
   weak[:max_sections] cap. 3.5 sorts organic tier-0 sections ahead of
   downgraded ones -- a section that never had a false header signal is a better
   fallback candidate than an ex-TOC entry.

2. **Shared ranking helper / --diagnose drift fix.** The downgrade + ordering
   logic is extracted into _rank_candidate_sections, used by BOTH
   build_candidate_snippets and diagnose_section_candidates. In 3.4 the
   diagnostic recomputed selection without the downgrade, so --diagnose could
   report SELECTED/dropped labels that disagreed with what was actually sent to
   the model. Diagnostics now also label demoted sections tier0(toc-downgraded).

## Effect vs 3.4 miss set

- **#47 (AzulSa) fixed** -- the designed target. In 3.4 a downgraded TOC-style
  section surfaced first and only 320 chars were sent; with the ordering fix the
  preamble section gets in (4,224 chars sent) and "January 6th, 2016" is
  correctly extracted.
- **#38 (SouthernStar) flipped OK->MISS on model variance, not the change.**
  Identical snippet selection in both runs (22,721 chars, 6,103 input tokens);
  the model returned the term clause in the 3.4 run and abstained in this one.
  It scores OK again in the 3.6 run with the same selection.
- Remaining misses (#6, 7, 10, 11, 24, 25, 26, 40, 43, 44, 48) were diagnosed
  with --diagnose: for most, the gold text IS inside a selected snippet but
  CUAD's gold is an "effective upon signing"-style clause while the model
  reports the preamble date -- a prompt/interpretation problem addressed in 3.6.
  #40/#43/#48 are retrieval gaps (pattern coverage; a 684-char caption section
  that dodges the 400-char downgrade threshold).
